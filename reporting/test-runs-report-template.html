Legend

Library test statuses:

  • T stands for Timeout. The test suite had probably hang.
  • C stands for Crash. The lisp process running the test suite terminated without returning test status.
  • F stands for FAIL
  • K stands for Known FAIL - test failures exist, but all the failures are already known for the library developers.
  • O stands for OK
  • U stands for Unexpected OK - all the tests succeeded, but there are tests marked by the library developers as "known failure", therefore absense failures is a problem deserving to be reported to the developers
  • R stands for No Resourse - the test suite can not be run because some required resource is absent in the environment. For example, CFFI library test suite needs a small C library compiled to DLL. User must do it manually. In case the DLL is absent, the LIBTEST method for CFFI returns :NO-RESOURCE

In the result table you may click the status letter to open the library test logs
